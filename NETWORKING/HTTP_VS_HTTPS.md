# HTTP vs HTTPS

## 🌐 HTTP (HyperText Transfer Protocol)

HTTP is a protocol used for communication between a client (browser) and a server. It transfers data in plain text, which makes it insecure.

### Key Points:
- No encryption
- Data is sent in plain text
- Vulnerable to interception
- Uses port 80

---

## 🔐 HTTPS (HyperText Transfer Protocol Secure)

HTTPS is the secure version of HTTP. It uses SSL/TLS encryption to protect data during transmission.

### Key Points:
- Encrypted communication using SSL/TLS
- Secure data transfer
- Prevents data interception
- Uses port 443
- Trusted by browsers (lock icon 🔒)

---

## ⚖️ HTTP vs HTTPS

| Feature | HTTP | HTTPS |
|----------|------|-------|
| Security | Not secure | Secure |
| Encryption | No | Yes (SSL/TLS) |
| Data Format | Plain text | Encrypted |
| Port | 80 | 443 |
| Trust Level | Low | High |

---

## 🌍 Example

HTTP is like sending a postcard that anyone can read, while HTTPS is like sending a sealed envelope that only the receiver can open.

---

## 🧠 Interview Answer (One Line)

HTTP is an unsecured protocol for data transfer, while HTTPS is a secure version of HTTP that uses SSL/TLS encryption to protect data.
